## ✅ ตรวจหลังติดตั้ง (เปิดใน Incognito กัน cache)
ทดสอบทีละหน้า แล้วเช็คว่า "ข้อมูลเข้า Supabase ไม่ไป V1":

- **stock-withdraw** เบิกของ 1 รายการ → ดูใน Supabase ตาราง `stock_withdraw` มีแถวใหม่
- **stock-receive** รับเข้า + แนบบิล → `stock_receive` มีแถว + `receipt_url` มีลิงก์รูป
- **stock-manage** เพิ่มรายการใหม่ (ดู item_id รันต่อ เช่น W045) / แก้ค่าขั้นต่ำ / กดปิดใช้งาน
- **stock-audit** นับจริง → `stock_audit` มีแถว · ลองติ๊ก "ปรับยอด" 1 รายการบนของทดสอบ แล้วเช็คคงเหลือขยับตาม ⚠️ (ดูหมายเหตุล่าง)
- **stock-close** ปิดรอบ → `stock_daily` มีแถวครบทุกรายการ (open/recv/withdraw/balance/used/diff)
- **attend** ตอกบัตรเข้า–ออก → `attendance` มีแถว + รูปขึ้นใน bucket `attendance`

ลบแถวทดสอบออกได้หลังเช็คเสร็จ

## ⚠️ หมายเหตุสำคัญ
- **ออดิทแบบ "ปรับยอด"**: ตอนติ๊ก "ปรับคงเหลือให้ตรงนับจริง" โค้ดจะเขียน movement แก้ส่วนต่าง
  (รับเข้า ถ้านับจริงมากกว่า / เบิก ถ้านับจริงน้อยกว่า) ใส่โน้ต "ปรับยอดจากออดิท"
  → คงเหลือจะขยับถูก แต่ยอด "รับเข้า/เบิก" ในหน้าวิเคราะห์จะรวมรายการปรับนี้ด้วย
  **ลองกับของทดสอบก่อน** ถ้าอยากให้ปรับแบบไม่กระทบสถิติ บอกได้ จะเปลี่ยนเป็นเขียน snapshot แทน
- **ปิดรอบยังไม่ส่ง LINE** (รอ Edge Function) — ปุ่มยังเขียนว่าส่ง LINE แต่จะข้ามไปก่อน
- สาขาในสต๊อกตั้งค่าตายตัวไว้ `'Pantip Ngamwongwan'` (ตัวแปร `SB_STOCK_BRANCH` บนหัวบล็อกใหม่ใน shared.js) — ร้านเดียวพอ
